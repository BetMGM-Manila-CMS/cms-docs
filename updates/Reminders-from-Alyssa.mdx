Hi everyone,

Just a couple of quick reminders to help streamline our promo build process and improve visibility across teams:

### 1. Always Include Rest IDs in All Relevant WF Tickets

When creating a shared promotion—even if only one restaurant ID (rest ID) is involved—please make sure that rest ID is included in all applicable builds and comments within Workflow (WF).

Why this matters:
OPS need to quickly locate specific rest IDs when reviewing tickets. If the ID is only listed in one place, it can slow things down and create unnecessary back-and-forth.

### 2. Build Static Tiles for All Thunderbite Promotions

For any Thunderbite promotions, please ensure that static tiles are always created.

Why this matters:
Since Thunderbite promos do not go through PAT, a static tile is always required for proper execution and visibility.
